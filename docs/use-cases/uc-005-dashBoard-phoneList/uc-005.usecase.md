## Use Case Description

## User story
As a staff member on duty, I want to see which fixed work phones are in use and assign my name to the phone number I carry for my selected shift, so that other staff can identify who has each phone during the current working shift.

### Brief Use Case
**Title**: Dashboard PhoneList  
**Success Flow**:  
The staff member selects a shift type, views the list of fixed work phone numbers, and assigns their name to the number they carry. The assignment is saved automatically when the staff member leaves the input field, and the dashboard reflects the current assignments for the selected shift.

### CaseCasual Use Case
**Title**: Dashboard PhoneList  
**Scope**: Slottets Drifttavlen (Dashboard)  
**Level**: User goal  
**Actors**:  
- System Time Event  
- Staff member  

**Main Flow (Dashboard refresh)**:  
1: System Time Event occurs (every 60 seconds).
2: Dashboard fetches the current phone assignments for the active shift.
3: Dashboard displays the updated PhoneList assignments.

**Main Flow (Assign name to phone number)**:  
1: Staff member opens the dashboard PhoneList view.
2: System displays the fixed phone numbers (41522, 41523, 41524, 41525, 41526, 41527, 41529) and any current name assignments for the active working shift.
3: Staff member selects a shift type (day/evening/night).
4: System updates the list to show assignments for the selected shift.
5: Staff member enters their name in the input field next to the phone number they carry.
6: Staff member leaves the input field.
7: System automatically saves the assignment for the selected shift.
8: System displays the updated assignment.

**Main Extensions**:  
&nbsp;&nbsp;&nbsp;&nbsp;3a: Staff member has not selected a shift type.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: System prevents assignment editing.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: System prompts the staff member to select a shift type before assigning a name.

&nbsp;&nbsp;&nbsp;&nbsp;5a: Staff member clears the name field (removes assignment).
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1: Staff member deletes the name and leaves the input field.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2: System automatically saves the empty value as “unassigned” for the selected shift.

**Summary**: Staff select a shift, view the fixed work phone numbers, and assign (or clear) a name per phone for the selected working shift. Saving happens automatically on leaving the input field.

### Fully Dressed Use Case
**Title**: Dashboard PhoneList  
**Scope**: Slottets Drifttavlen (Dashboard)  
**Level**: User goal  
**Actors**:  
- Primary: System Time Event  
- Secondary: Staff member  
- Supporting: Dashboard system  

**Related Requirements**:  
- PhoneList is not a contact list for making calls.
- Phone numbers are fixed and never change.
- Assignment is per working shift: day/evening/night.
- Staff must select a shift type before assigning a name.
- Assignment is saved automatically when leaving the input field.

**Preconditions**:  
- Staff member has access to the dashboard.
- The system is configured with the fixed phone numbers: 41522, 41523, 41524, 41525, 41526, 41527, 41529.

**Main Flow**:  
1. Staff member navigates to the dashboard section containing PhoneList.
2. System displays the list of fixed phone numbers and the current name assignment for each number for the currently active working shift context (if any).
3. Staff member selects a shift type (day, evening, night).
4. System refreshes the PhoneList to show assignments for the selected shift.
5. Staff member chooses the phone number they carry.
6. Staff member enters their name in the assignment field for that phone number.
7. Staff member leaves the input field.
8. System automatically saves the assignment (phone number + staff name + selected shift).
9. System displays the updated assignment for the selected shift.

**Separate Flow: Automatic refresh (System Time Event)**:
1. System Time Event occurs (every 60 seconds).
2. Dashboard fetches the current phone assignments for the active shift.
3. Dashboard updates the PhoneList display.

**Extensions**:

&nbsp;&nbsp;&nbsp;&nbsp;3a. Shift type not selected (from step 3)
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System disables assignment fields (read-only) until a shift type is selected.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2. System displays a message requesting the staff member to select a shift type.

&nbsp;&nbsp;&nbsp;&nbsp;6a. Staff member removes an existing assignment (from step 6)
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. Staff member clears the assignment field content.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2. Staff member leaves the input field.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3. System automatically saves the removal and marks the phone number as unassigned for the selected shift.

&nbsp;&nbsp;&nbsp;&nbsp;8a. Save fails due to connectivity or server error (from step 8)
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System keeps the entered value in the UI.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2. System displays an error indicating the assignment could not be saved.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3. System allows the staff member to retry by leaving the field again or performing an explicit retry action if offered.

**Postconditions**:  
- The selected shift has updated assignments for one or more phone numbers.
- The PhoneList view reflects the persisted assignments for the selected shift.
