## Operation Contract

### Operation
`refreshPhoneListEvery60Seconds()`

### Actor
System Time Event

### Scope
Dashboard view (automatic refresh of PhoneList every 60 seconds)

### Preconditions
- Dashboard is running.
- PhoneList is visible.

### Postconditions
- Dashboard displays updated phone assignments for the active shift.

### Inputs
- None

### Outputs
- Updated PhoneList displayed on the dashboard.

### Stakeholders and Interests
- Staff member: sees an up-to-date PhoneList without manual refresh.

### Success Guarantee
- The dashboard view reflects the latest persisted phone assignments for the active shift.

### Minimal Guarantee
- If refresh fails, the dashboard continues to show the last successfully displayed PhoneList.

### Exceptions
- Refresh fails due to connectivity or server error: dashboard shows last known PhoneList and may display an error indicator.
