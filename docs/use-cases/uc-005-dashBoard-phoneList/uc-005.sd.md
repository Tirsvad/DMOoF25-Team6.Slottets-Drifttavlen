## Sequence Diagram: Dashboard PhoneList (Automatic Refresh)

### Presentation Layer → Application Layer
```mermaid
sequenceDiagram
    actor SystemTimeEvent as System Time Event
    participant PhoneListDashboard as Presentation Layer
    participant PhoneAssignmentService as Application Layer

    SystemTimeEvent->>+PhoneListDashboard: TriggerRefresh(60s)
    PhoneListDashboard->>+PhoneAssignmentService: GetCurrentPhoneAssignmentsForActiveShift()
    PhoneAssignmentService-->>-PhoneListDashboard: PhoneAssignmentDto[]
    PhoneListDashboard-->>-PhoneListDashboard: RenderUpdatedPhoneList(PhoneAssignmentDto[])

    alt NoActiveShift
        PhoneAssignmentService-->>PhoneListDashboard: EmptyResult()
    else DataAccessError
        PhoneAssignmentService-->>PhoneListDashboard: Error(message)
    end
```

### WebApi Layer → Infrastructure Layer (Data Access)
```mermaid
sequenceDiagram
    participant PhoneAssignmentService as Application Layer
    participant PhoneAssignmentManager as Infrastructure Layer
    participant WebApi as WebApi

    PhoneAssignmentService->>+PhoneAssignmentManager: GetCurrentPhoneAssignmentsForActiveShift()
    PhoneAssignmentManager->>+WebApi: GET /api/phoneassignments/active-shift
    alt Success
        WebApi-->>PhoneAssignmentManager: 200 OK (PhoneAssignmentDto[])
        PhoneAssignmentManager-->>PhoneAssignmentService: PhoneAssignmentDto[]
    else Error
        WebApi-->>PhoneAssignmentManager: 4xx/5xx (Error)
        PhoneAssignmentManager-->>PhoneAssignmentService: Error(message)
    end
```

## Notes
- Scope: only the automatic refresh of PhoneList every 60 seconds, triggered by `System Time Event`.
- The dashboard requests current PhoneAssignments for the active shift.
- Data access uses WebAPI and is abstracted by the Infrastructure `PhoneAssignmentManager`.
- Returned data is represented as DTOs (`PhoneAssignmentDto[]`) across boundaries.
- `PhoneAssignment` domain concept includes `PhoneNumber` and `ShiftType`.
- Clean Architecture dependency direction is preserved (Infrastructure → Application → Domain).
