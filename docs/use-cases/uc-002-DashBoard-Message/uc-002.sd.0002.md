## Sequence Diagram
```mermaid
sequenceDiagram
    actor Employee as Actor
    participant WebUI as WebUI
    participant MessageController as MessageController
    participant MessageService as MessageService
    participant MessageRepository as MessageRepository

    Employee->>+WebUI: selectResident(residentId)
    WebUI->>+MessageController: getMessages(residentId)
    MessageController->>+MessageService: getMessagesForResident(residentId)
    MessageService->>+MessageRepository: fetchMessages(residentId)
    MessageRepository-->>-MessageService: messages
    MessageService-->>-MessageController: messages
    MessageController-->>-WebUI: messages
    WebUI-->>-Employee: showMessages(messages)

    Employee->>+WebUI: addMessage(residentId, messageText)
    WebUI->>+MessageController: addMessage(residentId, messageText)
    MessageController->>+MessageService: addMessage(residentId, messageText)
    MessageService->>+MessageRepository: saveMessage(residentId, messageText)
    alt addMessage success
        MessageRepository-->>MessageService: messageSaved
        MessageService-->>MessageController: success
        MessageController-->>WebUI: confirmMessageAdded()
        WebUI-->>Employee: confirmMessageAdded()
    else addMessage error
        MessageRepository-->>-MessageService: error
        MessageService-->>-MessageController: error
        MessageController-->>-WebUI: errorMessage("Message cannot be saved")
        WebUI-->>-Employee: errorMessage("Message cannot be saved")
    end

    Employee->>+WebUI: editMessage(residentId, messageId, newText)
    WebUI->>+MessageController: editMessage(residentId, messageId, newText)
    MessageController->>+MessageService: editMessage(residentId, messageId, newText)
    MessageService->>+MessageRepository: updateMessage(residentId, messageId, newText)
    alt editMessage success
        MessageRepository-->>MessageService: messageUpdated
        MessageService-->>MessageController: success
        MessageController-->>WebUI: confirmMessageEdited()
        WebUI-->>Employee: confirmMessageEdited()
    else editMessage error
        MessageRepository-->>-MessageService: error
        MessageService-->>-MessageController: error
        MessageController-->>-WebUI: errorMessage("Message cannot be saved")
        WebUI-->>-Employee: errorMessage("Message cannot be saved")
    end

    Employee->>+WebUI: deleteMessage(residentId, messageId)
    WebUI->>+MessageController: deleteMessage(residentId, messageId)
    MessageController->>+MessageService: deleteMessage(residentId, messageId)
    MessageService->>+MessageRepository: deleteMessage(residentId, messageId)
    alt deleteMessage success
        MessageRepository-->>MessageService: messageDeleted
        MessageService-->>MessageController: success
        MessageController-->>WebUI: confirmMessageDeleted()
        WebUI-->>Employee: confirmMessageDeleted()
    else deleteMessage error
        MessageRepository-->>-MessageService: error
        MessageService-->>-MessageController: error
        MessageController-->>-WebUI: errorMessage("Message cannot be deleted")
        WebUI-->>-Employee: errorMessage("Message cannot be deleted")
    end
```

**Note:** While Strict UML purists argue that actor is not part of sequence diagram, we can use actor in sequence diagram if it helps to clarify the interactions and roles of different participants in the system. The key is to ensure that the diagram remains clear and easy to understand for all stakeholders even it breaks strict UML rules.

---

**Note on DTOs and Data Transformation:**
- Data Transfer Objects (DTOs) are required between WebUI and backend layers (MessageController, MessageService) to decouple UI models from domain models.
- Example: `addMessage(residentId, messageText)` in WebUI is transformed into an `AddMessageDto` when sent to the controller/service layer.
- Data returned from the repository is mapped to DTOs before being sent to the WebUI.
- All data transformations should be explicit and documented in the implementation.

**DTO Example:**
```csharp
public class AddMessageDto
{
    public int ResidentId { get; set; }
    public string MessageText { get; set; }
}
```
