## Use Case Beskrivelse

## Brugerhistorie
Som medarbejder vil jeg kunne logge ind med mine legitimationsoplysninger, så jeg sikkert kan få adgang til dashboardet og beboeroplysninger.

### Kort Use Case
**Titel**: Brugerlogin  
**Succesflow**:  
Medarbejderen åbner login-siden, indtaster gyldigt brugernavn og adgangskode, og systemet giver adgang til dashboardet.

### Casual Use Case
**Titel**: Brugerlogin  
**Scope**: Autentifikation  
**Level**: User Goal  
**Aktører**:
- Medarbejder
- Autentifikationssystem

**Main Flow**:
1. Medarbejderen åbner login-siden.
2. Medarbejderen indtaster brugernavn og adgangskode.
3. Autentifikationssystemet validerer legitimationsoplysningerne.
4. Systemet opretter en autentificeret session.
5. Systemet viderestiller medarbejderen til dashboardet.

**Main Extensions**:
2a. Brugernavn eller adgangskode mangler.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet viser valideringsbesked.  
3a. Legitimationsoplysninger er ugyldige.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet viser login-fejlmeddelelse.  
3b. Kontoen er låst.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet informerer om, at kontoen er låst.  
4a. Autentifikationstjenesten er utilgængelig.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet viser midlertidig fejlmeddelelse.  
**Summary**: Medarbejderen logger sikkert ind og får adgang til applikationens funktioner ved gyldig konto.  

### Fully Dressed Use Case
**Titel**: Brugerlogin  
**Scope**: Autentifikation  
**Level**: User Goal  
**Aktører**:  
- Medarbejder
- Autentifikationssystem

**Relaterede Krav**:
- Sikker brugerautentifikation
- Adgangskontrol til dashboard og beboerdata

**Forudsætninger**:
- Medarbejderkontoen findes og er aktiv.
- Login-siden er tilgængelig.

**Hovedflow**:
1. Medarbejderen åbner login-siden.
2. Medarbejderen indtaster brugernavn og adgangskode.
3. Autentifikationssystemet validerer legitimationsoplysningerne.
4. Systemet opretter en autentificeret session.
5. Systemet viderestiller medarbejderen til dashboardet.

**Udvidelser**:  
a. På ethvert tidspunkt, hvis en teknisk fejl opstår, viser systemet en generisk fejl og forbliver på login-siden.  
2a. Brugernavn eller adgangskode mangler.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet viser valideringsbesked og beder om nødvendige input.  
3a. Legitimationsoplysninger er ugyldige.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet viser login-fejlmeddelelse og tillader nyt forsøg.  
3b. Kontoen er låst.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet informerer om, at kontoen er låst, og afviser login.  
4a. Autentifikationstjenesten er utilgængelig.  
&nbsp;&nbsp;&nbsp;&nbsp;1. Systemet viser midlertidig fejlmeddelelse og beder medarbejderen prøve senere.  

**Postconditions**:  
- Ved succes: autentificeret session er aktiv, og medarbejderen er på dashboardet.
- Ved fejl: ingen autentificeret session oprettes.
