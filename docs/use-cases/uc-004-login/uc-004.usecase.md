## Use Case Description

## User story
As a caregiver, I want to log in with my credentials, so that I can securely access the dashboard and resident information.

### Brief Use Case
**Title**: User Login

**Success Flow**:

The caregiver opens the login page, enters valid credentials, and the system grants access to the dashboard.

### Casual Use Case
**Title**: User Login

**Scope**: Authentication

**Level**: User Goal

**Actors**:
- Caregiver
- Authentication System

**Main Flow**:
1. Caregiver opens the login page.
2. Caregiver enters username and password.
3. Authentication system validates the credentials.
4. System creates an authenticated session.
5. System redirects caregiver to the dashboard.

**Main Extensions**:  
&nbsp;&nbsp;&nbsp;&nbsp;2a. Username or password is missing.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays validation message.  
&nbsp;&nbsp;&nbsp;&nbsp;3a. Credentials are invalid.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays login error message.  
&nbsp;&nbsp;&nbsp;&nbsp;3b. Account is locked.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System informs caregiver that account is locked.  
&nbsp;&nbsp;&nbsp;&nbsp;4a. Authentication service is unavailable.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays temporary service error message.  

**Summary**: The caregiver logs in securely and gains access to application features based on a valid account.

### Fully Dressed Use Case
**Title**: User Login

**Scope**: Authentication

**Level**: User Goal

**Actors**:
- Caregiver
- Authentication System

**Related Requirements**:
- Secure user authentication
- Access control to dashboard and resident data

**Preconditions**:
- Caregiver account exists and is active.
- Login page is available.

**Main Flow**:
1. Caregiver opens the login page.
2. Caregiver enters username and password.
3. Authentication system validates the credentials.
4. System creates an authenticated session.
5. System redirects caregiver to the dashboard.

**Extensions**:  
&nbsp;&nbsp;&nbsp;&nbsp;a. At any time, if a technical error occurs, system displays a generic error and keeps caregiver on login page.  
&nbsp;&nbsp;&nbsp;&nbsp;2a. Username or password is missing.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays validation message and requests required input.  
&nbsp;&nbsp;&nbsp;&nbsp;3a. Credentials are invalid.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays login error message and allows retry.  
&nbsp;&nbsp;&nbsp;&nbsp;3b. Account is locked.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System informs caregiver that account is locked and denies login.  
&nbsp;&nbsp;&nbsp;&nbsp;4a. Authentication service is unavailable.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. System displays temporary service error message and allows retry later.  

**Postconditions**:
- On success: authenticated session is active and caregiver is on dashboard.
- On failure: no authenticated session is created.
