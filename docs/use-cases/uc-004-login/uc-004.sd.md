## Sequence Diagram

### Presentation Layer → Application Layer

```mermaid
sequenceDiagram
    actor Caregiver as Caregiver
    participant LoginPage as BlazorLoginPage
    participant LoginService as Application.LoginService
    participant NotificationHandler as Application.Common.NotificationHandler
    participant AuthManager as Infrastructure.Identity.AuthManager

    Caregiver->>+LoginPage: EnterCredentials(username, password)
    LoginPage->>+LoginService: LoginAsync(username, password)
    LoginService->>+AuthManager: AuthenticateAsync(username, password)
    alt Success
        AuthManager-->>LoginService: AuthResult(success, user)
        LoginService->>+AuthManager: GenerateJwtToken(user)
        AuthManager-->>-LoginService: JwtToken
        LoginService->>+NotificationHandler: Info("Login successful")
        NotificationHandler-->>-LoginService: Ack
        LoginService-->>LoginPage: LoginResult(success, jwtToken)
        LoginPage-->>Caregiver: RedirectToDashboard(jwtToken)
    else Failure
        AuthManager-->>-LoginService: AuthResult(failure)
        LoginService->>+NotificationHandler: Error("Login failed")
        NotificationHandler-->>-LoginService: Ack
        LoginService-->>-LoginPage: LoginResult(failure)
        LoginPage-->>-Caregiver: ShowLoginError()
    end
    Note over NotificationHandler,LoginPage: UI subscribes to NotificationHandler for Info/Error messages
```

### Application Layer → Infrastructure Layer (External Interfaces)

```mermaid
sequenceDiagram
    participant AuthManager as Infrastructure.Identity.AuthManager
    participant SignInManager as Microsoft.AspNetCore.Identity.SignInManager
    participant UserManager as Microsoft.AspNetCore.Identity.UserManager
    participant JwtSecurityTokenHandler as System.IdentityModel.Tokens.Jwt.JwtSecurityTokenHandler

    AuthManager->>+UserManager: FindByNameAsync(username)
    UserManager-->>-AuthManager: User
    AuthManager->>+SignInManager: PasswordSignInAsync(user, password, isPersistent, lockoutOnFailure)
    alt SignInSuccess
      SignInManager-->>-AuthManager: SignInResult (Success/Failed/LockedOut)
      AuthManager->>+JwtSecurityTokenHandler: CreateToken(user)
      JwtSecurityTokenHandler-->>-AuthManager: JwtToken
    else Inactive/LockedOut
      SignInManager-->>AuthManager: SignInResult (LockedOut)
    else SignInFailed
      SignInManager-->>AuthManager: SignInResult (Failed)
    end

```

### WebApi Layer → Application Layer (Authentication & JWT)

```mermaid
sequenceDiagram
    participant LoginApiController as WebApi.Controllers.LoginApiController
    participant AuthManager as Infrastructure.Identity.AuthManager
    participant UserManager as Microsoft.AspNetCore.Identity.UserManager
    participant SignInManager as Microsoft.AspNetCore.Identity.SignInManager
    participant JwtSecurityTokenHandler as System.IdentityModel.Tokens.Jwt.JwtSecurityTokenHandler

    LoginApiController->>+SignInManager: ValidateCredentials(username, password)
    SignInManager->>+UserManager: FindByNameAsync(username)
    alt SignInSuccess
      SignInManager-->>-AuthManager: SignInResult (Success/Failed/LockedOut)
      AuthManager->>+JwtSecurityTokenHandler: CreateToken(user)
      JwtSecurityTokenHandler-->>-AuthManager: JwtToken
    else Inactive/LockedOut
      SignInManager-->>AuthManager: SignInResult (LockedOut)
    else SignInFailed
      SignInManager-->>AuthManager: SignInResult (Failed)
    end
```

> **Note:** Login with WebApi is only used for WebAssembly clients.

#### Notes
- `AuthManager` abstracts the Identity framework and is responsible for all authentication logic, including JWT generation.
- `SignInManager` and `UserManager` are from `Microsoft.AspNetCore.Identity`.
- `JwtSecurityTokenHandler` is used to generate the JWT.
- `NotificationHandler` is responsible for publishing Info and Error messages to which the UI subscribes.
- No internal system calls are shown beyond the manager abstraction.
- DTOs are used for credential transfer between layers.
