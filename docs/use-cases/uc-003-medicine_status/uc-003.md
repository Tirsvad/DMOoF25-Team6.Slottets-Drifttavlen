## Use Case Description

## User story
As a system, I want to provide the medicine and painkiller administration status for residents, so that staff can see what has been given and when, for each shift and for the last 24 hours.

### Brief Use Case
**Title**: Medicine and Painkiller Status Overview
**Success Flow**:
TimeEvent requests the system for the medicine schedule for day, evening, and night shifts, and for timestamps and types of painkillers administered to residents in the last 24 hours. The system returns the requested information.

### CaseCasual Use Case
**Title**: Medicine and Painkiller Status Overview
**Scope**: Medicine Administration
**Level**: User Goal
**Actors**:
- TimeEvent
- System
**Main Flow**:
1: TimeEvent requests medicine schedule either for day, evening, or night shifts.
2: System returns medicine administration status for all residents, including timestamps and whether medicine was given.
3: TimeEvent requests painkiller schedule for the last 24 hours.
4: System returns painkiller administration types and next allowed time it can be given for each resident.
**Main Extensions**:
2a: If error loading medicine data, system displays error and retry option.
2b: If no medicine history found, system displays no records message.
2c: If painkiller type not recognized, system displays warning.
4a: If error loading painkiller data, system displays error and retry option.
4b: If no painkiller history found, system displays no records message.
4c: If painkiller type not recognized, system displays warning.
**Summary**: The system provides a complete overview of medicine and painkiller administration for residents, supporting safe and informed care decisions.
