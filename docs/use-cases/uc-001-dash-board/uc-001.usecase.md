## Use Case Description

## User Story
As a system time event, the dashboard updates to display the current status and notes for a resident, so caregivers have up-to-date information.

### Brief Use Case
**Title**: Update Resident Dashboard
**Success Flow**: The dashboard screen automatically updates every 60 seconds, displaying the resident's current status and notes.

### Casual Use Case
**Title**: Update Resident Dashboard
**Scope**: Dashboard Screen
**Level**: System
**Actors**:
- System Time Event
**Main Flow**:
1. Time event is triggered
2. Request is sent to update the resident dashboard
3. The system displays the resident's status and notes
**Main Extensions**:
2a. Resident does not exist in the system
    1. Display 'Resident not found' message
3a. System fails to load resident data
    1. Display error message and retry option

### Fully Dressed Use Case
**Title**: Update Resident Dashboard
**Scope**: Dashboard Screen
**Level**: System
**Actors**:
- System Time Event
**Related Requirements**:
- Resident status tracking
- Dashboard update interval logic
**Preconditions**:
- The application is started and the dashboard is available on screen 2
- The resident exists in the system
**Main Flow**:
1. Time event is triggered (every 60 seconds)
2. Request is sent to update the resident dashboard
3. The system displays the resident's status and notes
**Extensions**:
a. At any time, system fails to load resident data
    1. Display error message and retry option
2a. Resident does not exist in the system
    1. Display 'Resident not found' message

**Postconditions**:
- The caregiver has updated information about the resident
