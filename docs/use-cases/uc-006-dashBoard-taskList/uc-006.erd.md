```mermaid
erDiagram
    CAREGIVER {
        Guid CaregiverId PK
        string Name
    }
    TASK {
        Guid TaskId PK
        string Title
        string Description
        string Status
        DateTime DueTime
        Guid ShiftId FK
        Guid CaregiverId FK
    }
    SHIFT {
        Guid ShiftId PK
        string Type
        DateTime StartTime
        DateTime EndTime
    }
    CAREGIVER ||--o{ TASK : "assigned to"
    SHIFT ||--o{ TASK : "has"
```

## Notes
- PK = Primary Key, FK = Foreign Key.
- Each Task is assigned to a Caregiver and belongs to a Shift.
- Relationships and attributes are derived from the DCD and domain model.
- Status is a string for simplicity; in implementation, it may be an enum.
