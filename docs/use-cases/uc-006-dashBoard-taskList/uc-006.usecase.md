## Use Case Description

## User story
As a staff member, I want to view, update, and complete my assigned tasks for the current shift, so I can efficiently manage my work and ensure all responsibilities are fulfilled.

### Brief Use Case
**Title**: Dashboard TaskList  
**Success Flow**:  
The staff member opens the dashboard, views their list of assigned tasks for the selected shift, marks tasks as completed or updates their status, and the dashboard reflects the current state of all tasks in real time.

### CaseCasual Use Case
**Title**: Dashboard TaskList  
**Scope**: Slottets Drifttavlen (Dashboard)  
**Level**: User goal  
**Actors**:  
- Staff member
- Dashboard system
**Main Flow**:  
1: Staff member opens the dashboard and sees the TaskList.
2: System displays the list of tasks assigned for the current shift.
3: Staff member marks a task as completed or updates its status.
4: System saves the update and refreshes the TaskList.
**Main Extensions**:  
    2a: No tasks assigned for the shift.  
        1: System displays a message indicating no tasks are assigned.  
    3a: Staff member cannot complete a task due to missing information.  
        1: System prompts for required information before allowing completion.  
**Summary**: Staff view and manage their shift tasks directly from the dashboard, with real-time updates and feedback.

### Fully Dressed Use Case
**Title**: Dashboard TaskList  
**Scope**: Slottets Drifttavlen (Dashboard)  
**Level**: User goal  
**Actors**:  
- Primary: Staff member  
- Supporting: Dashboard system  
**Related Requirements**:  
- Tasks are assigned per shift and user.
- Task status updates are saved in real time.
- Only tasks for the current shift are shown.
**Preconditions**:  
- Staff member is logged in and has access to the dashboard.
- Tasks are assigned for the current shift.
**Main Flow**:  
1. Staff member navigates to the dashboard TaskList section.
2. System displays the list of assigned tasks for the current shift.
3. Staff member marks a task as completed or updates its status.
4. System saves the update and refreshes the TaskList.
**Extensions**:
    2a. No tasks assigned for the shift (from step 2)
        1. System displays a message indicating no tasks are assigned.
    3a. Staff member cannot complete a task due to missing information (from step 3)
        1. System prompts for required information before allowing completion.
    4a. Save fails due to connectivity or server error (from step 4)
        1. System keeps the entered value in the UI.
        2. System displays an error indicating the update could not be saved.
        3. System allows the staff member to retry.
**Postconditions**:  
- The TaskList reflects the current state of all tasks for the shift.
- Completed tasks are marked accordingly.
