## Operation Contract


### openDashboardView
- **Preconditions**: Caregiver is authenticated and authorized to access the dashboard.
- **Postconditions**: System displays the list of available tasks for the current shift. No tasks are assigned by default.


### claimTask
- **Preconditions**: Caregiver has access to the TaskList and selects an available (unassigned) task to claim.
- **Postconditions**: System assigns the selected task to the caregiver and updates the TaskList to reflect the new assignment.

### updateTaskStatus
- **Preconditions**: Caregiver has claimed a task and selects it to update status.
- **Postconditions**: System updates the status of the selected task and confirms the update to the caregiver.

### markTaskComplete
- **Preconditions**: Caregiver has claimed a task and selects it to mark as complete.
- **Postconditions**: System marks the task as completed and confirms the completion to the caregiver.

### displayNoTasksMessage
- **Preconditions**: Caregiver opens the TaskList view and no tasks are assigned for the current shift.
- **Postconditions**: System displays a message indicating that no tasks are assigned.

### promptForRequiredInformation
- **Preconditions**: Caregiver attempts to complete a task but required information is missing.
- **Postconditions**: System prompts the caregiver to provide the required information before allowing completion.

### displaySaveError
- **Preconditions**: System fails to save a task update due to connectivity or server error.
- **Postconditions**: System displays an error message and allows the caregiver to retry the operation.
